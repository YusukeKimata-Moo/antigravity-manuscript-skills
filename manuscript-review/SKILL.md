# 学術論文原稿レビュー・修正スキル

分子生物学の学術論文の原稿を精査・修正するためのスキル。
日本語で原稿を執筆し、最終的に英訳することを前提とする。

---

## 1. 原稿読み込みと全体構造の把握

### 1.1 ファイル構成の確認

- 原稿ディレクトリ内のすべてのセクションファイルを特定する
- 典型的な構成: Introduction, Results, Discussion, Methods, Supplemental Methods
- すべてのファイルを読み込み、全体像を把握する

### 1.2 セクション間の対応関係マッピング

各セクション間で以下の対応関係を確認する：

| 確認項目     | Introduction → Results                         | Results → Discussion                          | Methods ↔ Supplemental Methods                    |
| ------------ | ---------------------------------------------- | --------------------------------------------- | ------------------------------------------------- |
| トピック順序 | Introで予告した順序でResultsが展開されるか     | Resultsの順序でDiscussionが議論されるか       | 同一手法が矛盾なく記述されているか                |
| 用語の一貫性 | 同一概念に同一用語を使用しているか             | 同左                                          | 同左                                              |
| 情報の過不足 | Introで伏線を張った項目がResultsで回収されるか | Resultsで示した知見がDiscussionで議論されるか | Methodsの簡略記述がSupplementalで補完されているか |

---

## 2. セクション別チェックリスト

### 2.1 Introduction

- [ ] **パラグラフ構造**: 一般的背景 → 先行研究 → 未解決の問題 → 本研究の目的・アプローチ → 結果の要約 の順で構成されているか
- [ ] **トピック順序**: Resultsの展開順と一致しているか
- [ ] **最終パラグラフの役割**:
  - 研究目的・手法・対象が明確に述べられているか
  - 結果の要約は**抽象的な大枠にとどめ**、Resultsの具体的内容を先取りしすぎていないか
  - 意義づけがDiscussionの結論と**重複していないか**（Introは事実ベースに留め、解釈や考察はDiscussionに委ねる）

### 2.2 Results

- [ ] **セクション見出し**: 各セクションの内容を端的に表す見出しがついているか
- [ ] **手法の切り替え**: 異なる手法（例: 固定標本観察→ライブイメージング、生化学→イメージング）の切り替え時に接続語で明示しているか
- [ ] **定量データの提示**: 定性的記述だけでなく、定量解析の結果（有意差の有無等）が示されているか
- [ ] **対照・検証**: 独立した手法やマーカーによる再現性の確認への言及があるか
- [ ] **先行研究との比較**: 各セクション末尾で関連する先行研究との一致/相違を簡潔に述べているか
- [ ] **Figure参照の一貫性**: Figure番号が正しく、同一番号が意図せず重複していないか

### 2.3 Discussion

- [ ] **パラグラフ構成**: 方法論的貢献 → 個別知見の考察（Resultsの順に対応） → 知見の総合・機能的含意 → 機構的考察 → 展望 の流れが自然か
- [ ] **Resultsとの対応**: Resultsで示したすべての主要知見がDiscussionで議論されているか
- [ ] **論理の飛躍チェック**: Discussionで主張する内容がResultsのデータで裏付けられているか。データに示されていない事実を前提にしている場合、先行研究の引用で根拠を明示する
- [ ] **パラグラフ間の重複**: 同じ結論が複数パラグラフで繰り返されていないか。重複がある場合：
  - 一方を**問題提起**に留め、他方で**結論・展望**を述べる
  - 具体的な今後の実験計画は最も関連の深いパラグラフに集約する
- [ ] **新規情報の導入**: Introで触れていない概念や生物種をDiscussionで引用する場合、文脈の説明やデータの限界を明記する
- [ ] **最終パラグラフ**: 研究全体の意義 → 手法の汎用性 → 今後の展望、で締める

### 2.4 Methods / Supplemental Methods

- [ ] **整合性**: Methodsの記述がSupplemental Methodsの詳細と矛盾していないか
- [ ] **条件の正確性**: 実験条件・試薬名・機器設定・解析手法がセクション間で一致しているか

## 3. 文体チェック（英訳を見据えた修正）

原稿は日本語で執筆し、最終的に英訳することを前提とする。以下の点を確認する。

### 3.1 修正すべきパターン

| パターン                         | 問題点                              | 修正方針                                        |
| -------------------------------- | ----------------------------------- | ----------------------------------------------- |
| 「AことからBと考えられている」   | Aが長い場合、英語で一文が冗長になる | Aで文を切り「この知見から、B」と2文にする       |
| 「AことにBことが報告されている」 | 並列が長く英語で不自然になる        | 2文に分割                                       |
| 「AがBを及ぼしていない」         | 英語で主語が不明確になりやすい      | 「AはBを及ぼさない」（「は」で主語明示）        |
| 「〜場合と〜場合とがあった」     | 英語に対応する構文がない            | 「〜または〜として観察された」に簡潔化          |
| 長い修飾節が名詞に係る           | 関係代名詞節に変換しにくい          | 修飾節を簡略化するか文を分割する                |
| 「〜における〜の〜が〜された」   | 助詞の連鎖で主語・目的語が不明確    | 主語を「は」で明示し文構造を単純化              |
| 「〜を行った」の多用             | performed/conductedの繰り返し       | 動詞を直接使う（「解析を行った」→「解析した」） |

### 3.2 修正の原則

- **意味・論理は絶対に変更しない**
- **修正は最小限**にとどめる
- 文の分割、助詞の変更、修飾節の簡略化のみ許可
- 科学的内容の加除は行わない

---

## 4. 修正実行のルール

### 4.1 修正前に必ずユーザーに確認

- 投稿予定の**ジャーナルと論文形式**（Article, Letterなど）をはじめにユーザーに確認する
- 修正箇所と修正案を**表形式で一覧提示**する
- 各修正の**理由**を明記する
- ユーザーの承認後に実行する

### 4.2 修正の粒度

- 同一ファイル内の非連続な変更は `multi_replace_file_content` を使用
- 変更は**最小限の文字列置換**にとどめ、パラグラフ全体の書き換えは避ける
- 変更完了後に**変更サマリを表形式で報告**する

### 4.3 提案と実行の分離

- ユーザーが「検討せよ」「確認せよ」と指示した場合 → **分析結果のみ提示**、修正は実行しない
- ユーザーが「修正せよ」と指示した場合 → **修正案を提示し承認を得てから**実行する
- ユーザーが「反映」「OK」「はい」と応答した場合 → 承認された修正を実行する

---

## 5. 分析レポートのフォーマット

セクション間の整合性チェック結果は以下の形式で報告する：

```markdown
### ✅ 問題なし

| 確認項目             | 判定 |
| -------------------- | ---- |
| トピック順序の整合性 | ✅   |
| ...                  | ...  |

### ⚠️ 指摘事項（N点）

#### 1. 【カテゴリ】問題の要約

- **箇所**: ファイル名・行番号
- **現在の記述**: 該当箇所の引用
- **問題点**: 具体的な説明
- **提案**: 最小限の修正案
```
